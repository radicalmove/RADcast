    Help & Guide

    RADcast helps you turn spoken-word recordings into cleaner deliverables without overwriting the source audio saved in your project.

    Typical project flow

    1. Create or open the project that should own the source audio, the finished runs, and any captions.
    2. Upload a new recording or choose a saved project audio file so the workspace has one clear source for the next run.
    3. Decide whether this run needs enhancement, cleanup, captions, or all three together.
    4. Start processing and keep the page open while the progress panel updates stage, ETA, and the machine doing the work.
    5. Open Completed versions, play the result, and compare it with earlier runs before deciding what to keep or share.
    6. Download the audio and caption files you need, then rerun with different settings if you want another version.

    Note

    Projects keep the originals

    The uploaded source file stays in the project. Each processing run creates a new completed version instead of overwriting the original audio or earlier outputs.

    What the main controls affect

    • Output format changes the downloaded audio file type for the next run.
    • Don't enhance audio skips the RADcast Optimized enhancement pass and keeps the run focused on cleanup, captions, or format conversion.
    • Reduce silence and Remove umms and ahhs change pacing and spoken detail, so they deserve a listen-back before you publish.
    • Closed captions adds an SRT or VTT file to the finished version when caption output is selected.

    Tip

    Run small checks before the final export

    Tip: Turn on Don't enhance audio when you only want cleanup or captions without the RADcast Optimized pass.

    Use this path when you want a new listening version of the selected recording while keeping the saved source audio available for later reruns.

    Before you start a run

    1. Choose audio by dropping a file onto the upload area or by selecting a saved project audio file.
    2. Wait for the filename or audio preview so you know RADcast is pointing at the right source before you press the main action button.
    3. Pick the output format and any cleanup or caption options you need for this run.
    4. Check whether you want full enhancement or only processing with Don't enhance audio.
    5. Leave the page open once the run starts so queued, processing, and completion updates stay visible.

    Use the main process button as a status-aware control:

    • When Don't enhance audio is off, the button label stays Enhance audio because RADcast Optimized is part of the run.
    • When Don't enhance audio is on, the button label changes to Process audio to reflect the lighter path.
    • After you click it, the progress panel becomes the best source of truth for stage, helper routing, and ETA.
    • When the job finishes, open the newest entry in Completed versions to listen back and download what you need.

    Note

    Most start failures are selection problems

    If nothing is selected, the run cannot start. Confirm the current file name or saved-audio preview first, especially after switching projects or changing sources.

    Use the cleanup controls when the recording is understandable but too hesitant, too padded with pauses, or too full of filler sounds for the final audience.

    What each cleanup control does

    • Reduce silence shortens longer gaps between phrases to the threshold shown on the slider.
    • Remove umms and ahhs targets hesitation sounds and short filler runs that make the delivery feel less polished.
    • Running both together gives the strongest pacing change, so compare the finished version with the source before publishing.

    Suggested review routine

    1. Start with one cleanup toggle when you are not sure how aggressive the edit should feel.
    2. Use a shorter sample or an early test run to listen for clipped breaths, unnatural joins, or meaning changes.
    3. Raise or lower the silence threshold based on whether the speaker still sounds too slow or suddenly sounds rushed.
    4. Keep the original run in Completed versions so you can compare pacing decisions side by side.

    Tip

    Favor intelligibility over aggression

    Cleanup settings are most useful when they remove distraction without making speech sound edited. If in doubt, keep a little more pause and rerun later.

    RADcast can attach caption output to the same run so the processed audio and the caption file stay paired in the same completed version.

    Caption workflow

    1. Select SRT or VTT before you start processing.
    2. Run the job normally and wait for the finished version to appear in Completed versions.
    3. Open that completed version and download the caption file that matches the format you selected.
    4. Review names, jargon, punctuation, and timing against the final audio before you publish or upload captions elsewhere.

    When captions need extra attention

    • Fast speakers, overlapping voices, strong accents, and domain-specific terms increase the chance of wording corrections.
    • If you changed pacing with cleanup, read the caption file against the processed result rather than the original recording.
    • If you do not choose SRT or VTT before the run, RADcast will finish the audio without creating a caption file.

    Note

    Caption files still need a human pass

    Captions are generated from the reviewed final audio path, but they are still draft deliverables until someone checks wording and timing.

    Trim clip is intended to narrow the working section before processing while leaving the saved project audio untouched.

    What trim is for

    • Use trim when you only need one section of a longer recording for the next processing run.
    • The goal is a non-destructive working range, not cutting or replacing the saved source file in the project.
    • Any captions, cleanup, or enhancement would then run against the trimmed working section rather than the full source.

    When trim controls are available

    1. Load or preview the selected source audio first so the workspace has a clear file context.
    2. Set the working start and end points for the section you want to keep for the next run.
    3. Process normally, then compare the trimmed result with the full-source version if you need to confirm the cut points.

    Troubleshooting

    Rollout awareness

    If trim controls are not visible in your current rollout, process the full file and rerun from the original project audio when trim becomes available.

    Helper processing lets another machine claim enhancement work when a helper app is online, which is useful for longer jobs or shared processing setups.

    How to use helpers safely

    1. Check the helper status pill before starting a long run so you know whether another machine is available.
    2. Use Set up helper app when you need install links or setup commands for the helper machine.
    3. Start processing and watch the progress details to see whether the job stays on the RADcast server or moves to a helper.
    4. If no helper is available in time, RADcast falls back to the server path instead of forcing you to start over.

    What the helper status means in practice

    • Connected means a helper recently checked in and can claim work if it stays online.
    • Not connected means the helper path is unavailable right now, so the server path should handle the run instead.
    • Checking usually means the page is refreshing helper state or has not yet received a fresh status response.

    Note

    Helpers are only useful when they stay awake

    A helper must stay online and idle enough to claim work. If it disconnects mid-run, RADcast may switch the job back to the server.

    Use these checks when a job will not start, progress looks strange, helper routing is confusing, or the finished result still needs another pass.

    If a run will not start

    • Confirm a project is open and the source file name or saved-audio preview is visible.
    • Recheck the selected options after changing projects, because the workspace may still be restoring saved project settings.
    • If an upload appears to finish but no audio name shows in the workspace, choose the file again and wait for the filename or preview before processing.

    If the job feels slow or the result feels wrong

    1. Leave the page open and watch the stage and ETA before assuming the run is stuck.
    2. If the helper shows offline or unknown, refresh helper status and expect the server fallback path to handle the run instead.
    3. If captions look wrong, review the final audio first, then correct the caption file before you publish it.
    4. If the pacing feels too aggressive, rerun with weaker cleanup settings or switch off the filler-word pass.
    5. If you expected a shorter section but got the full recording, go back to the source selection and confirm whether trim controls are available in this rollout.

    Troubleshooting

    Fast triage checklist

    Start with three checks in order: project open, correct source visible, and expected helper status. Those three explain most failed starts and confusing reruns.
